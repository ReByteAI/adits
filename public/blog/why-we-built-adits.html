Why we built Adits

The chat-on-the-left, canvas-on-the-right pattern is right. The captive-AI part is wrong. Here's the seam we built.

When Claude Design shipped, it was the first AI product that felt right for editing files. You could circle a bit of a slide and say "make this clearer" and it would actually do it. No more nudging shapes by 4 pixels in PowerPoint. No more fighting with Photoshop's selection tool. Hand the work to the agent, get the change back.

We loved the pattern. We didn't love the implementation.

The problems with captive AI

Claude Design ships an assistant that lives inside Anthropic's product. Your file goes through their service. You get the model they pick, the billing they set, the format constraints they decide. If you already pay $20/month for Claude through your own account, you pay again to use it inside Claude Design. If next year a different model is better at a particular kind of edit, you can't switch — you wait for Anthropic to swap it in for you.

That's the captive-AI tax. You're not buying a tool. You're renting access to a tool, on terms you don't set.

The same pattern shows up across the AI-product space. Cursor for code, Figma AI for design, ChatGPT's canvas mode, every "AI document editor" you've seen pitched. They all bundle the agent with the surface. You can't run their UI against a different agent. You can't run their agent against a different UI. The seam is glued shut.

The pattern was right; the seam was wrong

The right thing about Claude Design is the workflow. Drop in a file. See it on a canvas. Talk to the file in plain English. Get a new file back. That's a real shift in how to work with documents — for people who don't want to learn complex software, and for people who do but don't want to do mechanical edits by hand.

The wrong thing is that the agent and the UI are the same thing. Pull them apart and you get a much better product.

That's what Adits is.

What we changed

Adits is the surface — the workspace, the file viewers, the prompt composer, the bench. The agent is whatever coding CLI is already on your machine: Claude Code, Codex, Gemini CLI. The CLI runs locally (in self-host mode) or on a per-project cloud VM (in hosted mode). Either way, the agent is something you choose, log into separately, and can swap.

The first time you sit down at Adits, three things should hit:

  • The model is your choice. Pick the executor in the composer. Adits sends your prompt to that CLI and lets the CLI handle the rest. Tomorrow's better model arrives via your CLI's update — not via our redeploy.
  • The files are your files. In self-host mode, every project is a folder on your disk. Hosted mode runs your project in a per-user VM you can leave whenever you want; the source code is open, the export path is plain files.
  • The surface keeps growing. Each file type ships its own viewer (PDFs, slides, images, audio, sketches), and adding new ones is a single component. The agent doesn't care about the file type — that's the surface's job.

Why open source

Open source isn't a marketing move. It's the only way the seam stays open.

If we shipped Adits as a closed product, we'd have an incentive to lock the agent slot eventually — favor our model, charge for switching, route traffic through our proxy. The whole point of the architecture would erode.

The OSS code at github.com/ReByteAI/adits is the real product. Hosted mode adds infrastructure on top — a managed VM, a billing seam, persistent state across devices — but the surface is the same. If we ever break the contract, you can fork the surface and keep going.

What's next

Public beta is live at adits.ai. Self-host is available now: clone the repo, run two commands, edit files locally with your own CLI.

The roadmap is roughly: more file types, more skills (production templates the agent can pick up — slide decks, resumes, one-pagers), tighter integration with the major coding CLIs, and the per-turn picker getting smarter about which model handles which kind of edit best. We want to keep the surface boring and the agent slot wide open.

If that resonates, give it a try. If it doesn't, the source is there — fork it and build the version you actually want.
